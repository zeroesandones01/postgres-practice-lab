# Submission Naming Convention

To keep answers unique per learner and easy to review, name each SQL answer file with this pattern:

```text
<department_code>_<employee_last_name>_exercise<nn>.sql
```

Examples:

- `eng_stone_exercise01.sql`
- `hr_cole_exercise02.sql`
- `fin_ward_exercise03.sql`

## Rules

1. Use lowercase letters.
2. Use underscores only (`_`), no spaces.
3. Use a 2-digit exercise number (`01`, `02`, `03`).
4. Keep one file per exercise answer.
5. Add optional notes in `notes.md` inside your folder.

## Recommended folder layout

```text
submissions/
  <github-username>/
    eng_stone_exercise01.sql
    eng_stone_exercise02.sql
    notes.md
```
